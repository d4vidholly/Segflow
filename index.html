Segflow is a collection of web tools designed to solve real-world problems in grassroots sports and leisure, helping players and organisers streamline their processes.

Each tool is based on research into specific user flows—identifying friction points and designing accessible, focused alternatives. The process behind these solutions moves from sketch to wireframe to live, functional app, combining service design thinking with front-end development. Together, they reflect a hands-on approach to building digital tools that make everyday systems faster and easier to use.

Boro Predictor

Web App to replace office football predictions and improve user flow, designed for mobile

Overview

For the past few seasons, I’ve taken part in a football predictor league run by a friend for fans of my team, Middlesbrough. While it’s all for fun, the process of submitting predictions via Excel spreadsheets had become frustrating for both participants and the organiser. I decided to explore a better way to run the game by designing and building a prototype web app that simplifies the experience—especially for less tech-savvy users.

Current Format – Key Issues

  • Reliance on Desktop Excel
    Participants had to download a spreadsheet, input scores, save the file, and email it back—introducing friction from the start and no mobile-friendly alternative.
  • Tech Literacy Gaps
    Some users didn’t have Excel, while others accidentally formatted scores as dates or submitted blank files, resulting in confusion and back-and-forth.
  • Manual Submissions
    One family member (who will not be named!) wrote all their scores on paper, photographed it, and texted it to me—only to miss two fixtures. She still beat me in the final table.

Design Challenge

I set out to build a tool that:

  • Simplifies the Experience – Make it easy for everyone to enter predictions—especially those with limited digital experience.
  • Adds Enjoyment – Move beyond spreadsheets to something more visually engaging and interactive.
  • Creates Value – With all predictions captured centrally, there's potential for data visualisation and storytelling.

Importantly, the prototype still exports an Excel file to maintain compatibility with the current system, but this can evolve with organiser input.

Design Approach

  • Wireframes
    I began by sketching the user journey and basic screens, focusing on a smooth experience. This began as desktop-first but quickly developed into a mobile-friendly flow.
  • User Flows
    Tested edge cases—scores are saved locally in case of a crash, and a “Clear All” button was added with a reset confirmation to avoid accidental loss.

Key Features

  • Season Summary Tool
    After submitting predictions, users get a snapshot of how their season might play out. It highlights their overall tone—optimistic or pessimistic—compared to realistic outcomes, helping players imagine their season unfolding and adding narrative to what was once a static task.
  • Fixture Colour Coding
    Visual cues change fixture colours based on predicted results (win, draw, loss), making it easier to interpret at a glance.
  • Records Awareness (Planned Feature)
    The app could flag if a prediction breaks a historical record—e.g. “10–0 vs Sunderland would be our biggest ever league win.”

User Feedback

“I would definitely use this for next season’s challenge.”
“Love the visual colour change of the fixtures based on the result I predicted.”
“The season report is a great overview of my predictions—definitely too optimistic!”

Next Steps

  • Add admin tools for compiling results and managing leaderboards
  • Explore real-time prediction comparisons and historical stats
  • Integrate with external fixture APIs for live updates
  • Review whether Excel remains the best database format going forward

Wembley Board Game Companion

Board game companion app designed for desktop

Overview

I researched, designed, and built a modern web app version of the Wembley 2016 board game using HTML, CSS, and JavaScript. The goal was to preserve the nostalgic value of the original while solving long-standing usability and relevance issues. The live site replaces outdated, often misplaced physical components with a dynamic, customisable digital experience.

Original Game – Key Issues

Complexity: Cumbersome rules and fiddly mechanics (especially money distribution) slowed down gameplay and frustrated players.

Dependence on Physical Components: The game relied on custom dice and hard-to-find rule books—making it unplayable if anything was missing.

Unfair Mechanics: Dice outcomes felt unbalanced, and team strength lacked transparency, affecting fairness.

Outdated Elements: Team lineups became obsolete quickly due to promotion and relegation in real-world football.

Design Approach

User Experience First: Created wireframes focused on how players choose teams—prioritising alphabetical ordering, visual clarity, and intuitive layout.

Modern Interface: Drawing inspiration from football apps and live broadcasts, I introduced updated club badges, player ratings, and a clean, football-fan-friendly interface.

Enhanced Gameplay Context: Key gameplay variables—like international stars, game rules, and available funds—are clearly presented before kickoff to reduce confusion mid-game.

Customisable Mechanics: Dice rolls, team selection, and gameplay elements are now fully customisable, allowing for up-to-date seasonal versions and flexible rule sets.

Player Feedback

“Love the modern feel, especially the updated club badges.”
“Made playing so much easier—no more fumbling with money or dice.”
“Feels like waiting for scores on the telly again—nostalgic but smoother!”

Next Steps

  • Develop themed visual "skins" for earlier and newer seasons
  • Explore seasonal data integration for real-time updates
  • Investigate multiplayer and AI-driven gameplay options
